    brimr-downloader

    Python GUI application that uses Selenium browser automation to batch download Excel files from the Blue Ridge Institute for Medical Research website, which publishes annual NIH funding rankings for medical schools, departments, and principal investigators.

    BRIMR provides invaluable funding data spanning nearly two decades, but every Excel file has to be downloaded individually, with 70+ files per year. Manually downloading even five years of data takes over an hour. brimr-downloader transforms that into a single organized download with automatic file categorization across nine folders (school rankings, department summaries, basic science, clinical departments, PI rankings, geographic analyses, and more), reducing an estimated 8 to 10 hours of manual clicking to about 13 minutes per multi-year pull.

    Cross-platform with native Downloads folder detection on Windows, macOS, and Linux. Automatic year detection on launch, headless mode for background runs, and a comprehensive timestamped log file per session for full auditability.

    brimr_downloader.py · simplified 
    # Iterate years, classify each file by URL pattern,
# route into one of nine destination folders.
for year in years_to_fetch:
    driver.get(f"https://brimr.org/{year}/")
    for href, filename in collect_xlsx_links(driver):
        bucket = classify(filename)        # one of nine categories
        target = DESTINATIONS[bucket]
        download(href, target, log=session_log)

  • 02 github.com/hihipy/25live-cleaner →

    25live-cleaner

    Python GUI utility that transforms raw 25Live room booking Excel exports into clean, analysis-ready datasets. 25Live is a widely used campus space scheduling platform whose raw exports require hours of manual cleaning before any analysis can begin.

    Reconstructs fragmented overnight bookings (where 25Live splits multi-day events across rows using "cont" tokens instead of real timestamps), normalizes inconsistent text fields with HTML entities and mixed casing, and performs smart deduplication that distinguishes exact duplicates from legitimate multi-location records.

    Generates a comprehensive machine-readable JSON audit trail documenting script identity (version, SHA-256 hash), every cleaning decision (cont token replacements, stitching groups, deduplication keep/drop reasons), and output metadata. Designed for non-technical facilities management staff with a tkinter GUI rather than a command line.

    stitcher.py · simplified 
    # Stitch fragmented overnight bookings. 25Live splits multi-day
# events across rows using a "cont" placeholder for the second
# row's start time. Walk the rows, stitch back to a real range.
for prev, curr in zip(rows, rows[1:]):
    if curr.start == "cont" and curr.event_id == prev.event_id:
        prev.end = curr.end
        curr.drop = True
        audit.log("stitched", prev.event_id, prev.start, prev.end)

    qualtrics-processing-pipeline

    Jupyter Notebook pipeline that transforms raw Qualtrics survey Excel exports into a complete, documented, analysis-ready package.     pbi-model-export

    Tabular Editor 2 C# script that exports a complete Power BI model snapshot to a single portable JSON file. Tables, columns, relationships, measures with DAX expressions, hierarchies, partitions with M queries, RLS roles, data sources, plus a compact summary section designed for AI consumption.

    Power BI model structure is normally locked inside the desktop application with no built-in way to export the full architecture to a portable, machine-readable format. This script enables model documentation, impact analysis, architecture review, and direct LLM-assisted analysis without manually describing model structure.

    Includes a measure-dependency detector built on compiled regex bracket matching that runs in linear time (the v1 version was quadratic), JSON-safe character escaping for DAX expressions and M queries with embedded special characters, and configuration for hidden objects, partition source expressions, and RLS roles. Saves as a Tabular Editor Custom Action for one-click access.

  • 05 github.com/hihipy/qualtrics-report-generator →

    qualtrics-report-generator

    Python utility that converts Qualtrics CSV and QSF exports into formatted, self-contained HTML reports. Built specifically for administrative data intake surveys (institutional reporting, compliance questionnaires, accreditation self-studies, rankings collection) where a small number of respondents each complete different sections and the goal is readable per-response review rather than aggregate statistics.

    When a QSF file is provided, the parser extracts complete survey definitions including question type metadata, choices with display labels, and answer column labels. A 30-entry type map translates Qualtrics' internal question type tuples (matrix Likert, multiple choice variations, sliders, side-by-sides) to internal display types. When no QSF is provided, the script infers structure from CSV column patterns.

    A 13-type value classification system formats each cell appropriately (URLs become links, file paths become download links with extracted filenames, coordinates display with degree symbols, JSON pretty-prints in collapsible blocks). The HTML output uses a colorblind-friendly palette based on Paul Tol's research, safe for all three common forms of color vision deficiency.

  • 06 github.com/hihipy/ai-csv-profiler →

    ai-csv-profiler

    Dual-interface Python utility (tkinter GUI and argparse CLI) that generates comprehensive, AI-optimized JSON profiles of CSV files. Built to produce structured metadata summaries that can be fed directly to LLM assistants (ChatGPT, Claude, etc.) to give them full structural awareness of a dataset before prompting, eliminating the need to paste raw data or manually describe column types.

    The CSV reader implements a multi-stage fallback: first iterating through eight encodings crossed with six separator candidates, then falling back to pandas auto-detection, and finally resorting to manual line-by-line parsing for severely corrupted files. Every operation is wrapped in defensive error handlers so the program never crashes on malformed data.

    The type detection engine classifies columns as numeric, boolean, datetime, categorical, or text using ordered priority rules (boolean patterns checked first to prevent misclassification), column-name keyword hints, a 90% numeric conversion threshold, and cardinality-based categorical-vs-text differentiation.

    # Multi-stage CSV reader. Eight encodings × six separators,
# then pandas auto-detect, then byte-level recovery.
for encoding, sep in product(ENCODINGS, SEPARATORS):
    try: return read_csv(path, encoding=encoding, sep=sep)
    except (UnicodeDecodeError, ParserError): continue
try: return read_csv(path, sep=None, engine="python")
except Exception: return manual_line_recovery(path)
